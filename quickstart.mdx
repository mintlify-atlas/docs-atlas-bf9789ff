## Setup FlexPrice

<Steps>
  <Step title="Clone the Repository">
    Clone the FlexPrice repository from GitHub:

    ```bash
    git clone https://github.com/flexprice/flexprice
    cd flexprice
    ```
  </Step>

  <Step title="Start All Services">
    Run the complete development environment setup with a single command:

    ```bash
    make dev-setup
    ```

    This command will:
    - Start all required infrastructure (PostgreSQL, Kafka, ClickHouse, Temporal)
    - Build the FlexPrice application image
    - Run database migrations and initialize Kafka
    - Start all FlexPrice services (API, Consumer, Worker)

    <Info>
      The setup process may take a few minutes on first run as it downloads Docker images and builds the application.
    </Info>
  </Step>

  <Step title="Verify Services are Running">
    Once setup is complete, verify all services are running:

    ```bash
    docker compose ps
    ```

    You should see the following services running:
    - `postgres` - PostgreSQL database
    - `kafka` - Apache Kafka message broker
    - `clickhouse` - ClickHouse analytics database
    - `temporal` - Temporal workflow engine
    - `temporal-ui` - Temporal web interface
    - `flexprice-api` - FlexPrice API server
    - `flexprice-consumer` - Event consumer service
    - `flexprice-worker` - Temporal worker service
  </Step>

  <Step title="Access the Services">
    Once running, you can access the following services:

    <CardGroup cols={2}>
      <Card title="FlexPrice API" icon="server">
        **http://localhost:8080**
        
        Main API endpoint for ingesting events and managing billing
      </Card>
      <Card title="Temporal UI" icon="clock">
        **http://localhost:8088**
        
        Monitor workflow executions and debug workflows
      </Card>
      <Card title="Kafka UI" icon="stream">
        **http://localhost:8084**
        
        View Kafka topics and messages (requires `--profile dev`)
      </Card>
      <Card title="ClickHouse" icon="database">
        **http://localhost:8123**
        
        Query analytics and event data
      </Card>
    </CardGroup>

    <Tip>
      To access Kafka UI, restart with the dev profile:
      ```bash
      docker compose --profile dev up -d kafka-ui
      ```
    </Tip>
  </Step>
</Steps>

## Ingest Your First Event

Now that FlexPrice is running, let's send your first usage event to track customer activity.

<Steps>
  <Step title="Get Your API Key">
    For local development, use the default API key:

    ```bash
    export API_KEY="0cc505d7b917e0b1f25ccbea029dd43f4002edfea46b7f941f281911246768fe"
    ```

    <Note>
      This is a development API key. In production, you'll generate unique API keys per environment.
    </Note>
  </Step>

  <Step title="Send a Usage Event">
    Send a test usage event using curl:

    <CodeGroup>
      ```bash curl
      curl -X POST http://localhost:8080/v1/events \
        -H "Content-Type: application/json" \
        -H "X-API-Key: 0cc505d7b917e0b1f25ccbea029dd43f4002edfea46b7f941f281911246768fe" \
        -d '{
          "customer_id": "customer_123",
          "event_name": "api_call",
          "timestamp": "'$(date -u +"%Y-%m-%dT%H:%M:%SZ")'",
          "properties": {
            "endpoint": "/api/v1/users",
            "method": "GET",
            "response_time_ms": 145
          }
        }'
      ```

      ```javascript JavaScript
      const response = await fetch('http://localhost:8080/v1/events', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'X-API-Key': '0cc505d7b917e0b1f25ccbea029dd43f4002edfea46b7f941f281911246768fe'
        },
        body: JSON.stringify({
          customer_id: 'customer_123',
          event_name: 'api_call',
          timestamp: new Date().toISOString(),
          properties: {
            endpoint: '/api/v1/users',
            method: 'GET',
            response_time_ms: 145
          }
        })
      });
      ```

      ```python Python
      import requests
      from datetime import datetime

      response = requests.post(
          'http://localhost:8080/v1/events',
          headers={
              'Content-Type': 'application/json',
              'X-API-Key': '0cc505d7b917e0b1f25ccbea029dd43f4002edfea46b7f941f281911246768fe'
          },
          json={
              'customer_id': 'customer_123',
              'event_name': 'api_call',
              'timestamp': datetime.utcnow().isoformat() + 'Z',
              'properties': {
                  'endpoint': '/api/v1/users',
                  'method': 'GET',
                  'response_time_ms': 145
              }
          }
      )
      ```

      ```go Go
      package main

      import (
          "bytes"
          "encoding/json"
          "net/http"
          "time"
      )

      func main() {
          event := map[string]interface{}{
              "customer_id": "customer_123",
              "event_name":  "api_call",
              "timestamp":   time.Now().UTC().Format(time.RFC3339),
              "properties": map[string]interface{}{
                  "endpoint":          "/api/v1/users",
                  "method":            "GET",
                  "response_time_ms": 145,
              },
          }

          body, _ := json.Marshal(event)
          req, _ := http.NewRequest("POST", "http://localhost:8080/v1/events", bytes.NewBuffer(body))
          req.Header.Set("Content-Type", "application/json")
          req.Header.Set("X-API-Key", "0cc505d7b917e0b1f25ccbea029dd43f4002edfea46b7f941f281911246768fe")

          client := &http.Client{}
          resp, _ := client.Do(req)
          defer resp.Body.Close()
      }
      ```
    </CodeGroup>

    <Success>
      If successful, you'll receive a response with the event ID and ingestion timestamp.
    </Success>
  </Step>

  <Step title="Verify Event Ingestion">
    Check that your event was processed successfully:

    **View in Kafka UI:**
    1. Navigate to http://localhost:8084
    2. Click on the `events` topic
    3. View the latest messages

    **Query in ClickHouse:**
    ```bash
    docker compose exec clickhouse clickhouse-client \
      --user=flexprice \
      --password=flexprice123 \
      --database=flexprice \
      --query="SELECT * FROM events ORDER BY timestamp DESC LIMIT 5"
    ```

    **Check API Logs:**
    ```bash
    docker compose logs -f flexprice-api
    ```
  </Step>
</Steps>

## Useful Commands

Here are some helpful commands for managing your local FlexPrice installation:

<CodeGroup>
  ```bash Restart Services
  # Restart only FlexPrice services (not infrastructure)
  make restart-flexprice
  ```

  ```bash Stop All Services
  # Stop all running containers
  make down
  ```

  ```bash Clean Start
  # Remove all data and start fresh
  make clean-start
  ```

  ```bash View Logs
  # API logs
  docker compose logs -f flexprice-api

  # Consumer logs
  docker compose logs -f flexprice-consumer

  # Worker logs
  docker compose logs -f flexprice-worker

  # All services
  docker compose logs -f
  ```

  ```bash Check Service Health
  # View status of all services
  docker compose ps

  # Check specific service health
  docker compose exec flexprice-api wget --spider http://localhost:8080/health
  ```
</CodeGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Services fail to start">
    Ensure Docker has enough resources allocated (at least 4GB RAM recommended).

    Check Docker daemon status:
    ```bash
    docker info
    ```

    View detailed logs:
    ```bash
    docker compose logs
    ```
  </Accordion>

  <Accordion title="Database connection errors">
    Check database service health:
    ```bash
    docker compose ps postgres clickhouse
    ```

    View database logs:
    ```bash
    docker compose logs postgres
    docker compose logs clickhouse
    ```

    Verify database is accepting connections:
    ```bash
    docker compose exec postgres pg_isready -U flexprice
    ```
  </Accordion>

  <Accordion title="Kafka connection issues">
    Verify Kafka is running:
    ```bash
    docker compose logs kafka
    ```

    List Kafka topics:
    ```bash
    docker compose exec kafka kafka-topics --bootstrap-server kafka:9092 --list
    ```

    Reinitialize Kafka topics:
    ```bash
    make init-kafka
    ```
  </Accordion>

  <Accordion title="Events not appearing in ClickHouse">
    Check consumer service logs:
    ```bash
    docker compose logs -f flexprice-consumer
    ```

    Verify Kafka consumer group status:
    ```bash
    docker compose exec kafka kafka-consumer-groups \
      --bootstrap-server kafka:9092 \
      --describe \
      --group flexprice-consumer-group
    ```

    Check ClickHouse for recent events:
    ```bash
    docker compose exec clickhouse clickhouse-client \
      --user=flexprice \
      --password=flexprice123 \
      --query="SELECT count() FROM flexprice.events"
    ```
  </Accordion>
</AccordionGroup>
