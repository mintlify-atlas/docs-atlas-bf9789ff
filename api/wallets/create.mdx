---
title: "Create Wallet"
api: "POST /v1/wallets"
description: "Create a prepaid or postpaid wallet for a customer to manage credits and usage-based billing."
---

## Request Body

<ParamField body="customer_id" type="string">
  The ID of the customer this wallet belongs to. Either `customer_id` or `external_customer_id` is required.
</ParamField>

<ParamField body="external_customer_id" type="string">
  The external customer ID from your system. Either `customer_id` or `external_customer_id` is required.
</ParamField>

<ParamField body="currency" type="string" required>
  Three-letter ISO currency code (e.g., USD, EUR, GBP)
</ParamField>

<ParamField body="wallet_type" type="string" default="PREPAID">
  Type of wallet
  
  **Values:**
  - `PREPAID`: Customer must load credits before usage
  - `POSTPAID`: Customer can use credits and pay later
</ParamField>

<ParamField body="conversion_rate" type="string" default="1">
  Conversion rate from wallet credits to currency during consumption
  
  Examples:
  - `1`: 1 credit = 1 USD
  - `2`: 1 credit = 0.5 USD (2 credits per dollar)
  - `0.5`: 1 credit = 2 USD (0.5 credits per dollar)
</ParamField>

<ParamField body="topup_conversion_rate" type="string">
  Conversion rate for top-ups to the currency. If not provided, uses `conversion_rate`.
  
  Examples:
  - `1`: 1 USD = 1 credit
  - `2`: 1 USD = 0.5 credits
  - `0.5`: 1 USD = 2 credits
</ParamField>

<ParamField body="initial_credits_to_load" type="string" default="0">
  Number of credits to load into the wallet upon creation (not currency amount)
</ParamField>

<ParamField body="initial_credits_expiry_date_utc" type="string">
  Expiry date for initial credits in ISO 8601 format (e.g., 2025-12-31T23:59:59Z)
  
  If not set, credits don't expire.
</ParamField>

<ParamField body="description" type="string">
  Optional description for the wallet
</ParamField>

<ParamField body="metadata" type="object">
  Custom metadata as key-value pairs
</ParamField>

<ParamField body="alert_settings" type="object">
  Alert configuration for low balance notifications
  
  <Expandable title="Alert settings object">
    <ParamField body="alert_enabled" type="boolean" default="false">
      Enable or disable alerts
    </ParamField>
    
    <ParamField body="threshold_type" type="string">
      Type of threshold
      
      **Values:** `PERCENTAGE`, `ABSOLUTE`
    </ParamField>
    
    <ParamField body="threshold_value" type="string">
      Threshold value (percentage or absolute amount)
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="auto_topup" type="object">
  Auto top-up configuration
  
  <Expandable title="Auto top-up object">
    <ParamField body="enabled" type="boolean" default="false">
      Enable automatic top-ups when balance is low
    </ParamField>
    
    <ParamField body="threshold" type="string">
      Balance threshold that triggers auto top-up
    </ParamField>
    
    <ParamField body="amount" type="string">
      Amount to add during auto top-up
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="price_unit" type="string">
  Price unit code to use for wallet creation. If provided, overrides `currency` and `conversion_rate` with values from the price unit.
</ParamField>

## Wallet Types Explained

<AccordionGroup>
  <Accordion title="Prepaid Wallet">
    A prepaid wallet requires customers to load credits before using your service.
    
    **Use cases:**
    - Pay-as-you-go models
    - Promotional credits
    - Preventing overages
    - Prepaid customer segments
    
    **Behavior:**
    - Usage is blocked when balance reaches zero
    - Credits must be added before additional usage
    - Suitable for usage-based pricing only
  </Accordion>
  
  <Accordion title="Postpaid Wallet">
    A postpaid wallet allows customers to use services and pay later through invoicing.
    
    **Use cases:**
    - Enterprise customers
    - Net payment terms
    - Monthly invoicing cycles
    - Customers with credit arrangements
    
    **Behavior:**
    - Balance can go negative
    - Usage tracked and invoiced periodically
    - Supports all price types (fixed, usage, tiered, etc.)
  </Accordion>
</AccordionGroup>

## Conversion Rate Examples

<Tip>
  Conversion rates allow you to decouple internal credit accounting from actual currency values.
</Tip>

**Example 1: 1:1 conversion**
```json
{
  "conversion_rate": "1",
  "currency": "usd"
}
```
Result: 1 credit = $1.00 USD

**Example 2: Bulk discount pricing**
```json
{
  "conversion_rate": "0.8",
  "currency": "usd"
}
```
Result: 1 credit = $0.80 USD (customer gets 25% more value)

**Example 3: Premium pricing**
```json
{
  "conversion_rate": "1.5",
  "currency": "usd"
}
```
Result: 1 credit = $1.50 USD
