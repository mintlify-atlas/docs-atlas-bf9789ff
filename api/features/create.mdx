---
title: "Create Feature"
api: "POST /v1/features"
description: "Use when defining a new feature or capability to gate or meter (e.g. feature flags or usage-based limits). Ideal for boolean or usage features."
---

## Request Body

<ParamField body="name" type="string" required>
  Human-readable name for the feature (e.g., "API Access", "Advanced Analytics")
</ParamField>

<ParamField body="lookup_key" type="string">
  Unique identifier for API lookups (e.g., "api_access", "advanced_analytics")
  
  Must be unique within the environment. Used for programmatic access.
</ParamField>

<ParamField body="type" type="string" required>
  Type of feature
  
  **Values:**
  - `BOOLEAN`: Simple on/off feature flag
  - `METERED`: Usage-based feature with consumption tracking
  - `STATIC`: Feature with a fixed configuration value
</ParamField>

<ParamField body="description" type="string">
  Detailed description of what this feature provides
</ParamField>

<ParamField body="meter_id" type="string">
  ID of an existing meter to use for METERED features
  
  Either `meter_id` or `meter` must be provided for METERED features.
</ParamField>

<ParamField body="meter" type="object">
  Meter configuration for METERED features (alternative to meter_id)
  
  <Expandable title="Meter object">
    <ParamField body="name" type="string" required>
      Name of the meter
    </ParamField>
    
    <ParamField body="event_type" type="string" required>
      Event type to track (e.g., "api_call", "storage_gb")
    </ParamField>
    
    <ParamField body="aggregation" type="string" required>
      How to aggregate usage
      
      **Values:** `SUM`, `COUNT`, `MAX`, `UNIQUE_COUNT`
    </ParamField>
    
    <ParamField body="field" type="string">
      Field to aggregate (required for SUM and MAX)
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="unit_singular" type="string">
  Singular unit name (e.g., "API call", "GB")
  
  Both `unit_singular` and `unit_plural` must be provided together.
</ParamField>

<ParamField body="unit_plural" type="string">
  Plural unit name (e.g., "API calls", "GBs")
  
  Both `unit_singular` and `unit_plural` must be provided together.
</ParamField>

<ParamField body="alert_settings" type="object">
  Alert configuration for usage thresholds
  
  <Expandable title="Alert settings object">
    <ParamField body="alert_enabled" type="boolean" default="false">
      Enable usage alerts
    </ParamField>
    
    <ParamField body="threshold_type" type="string">
      Type of threshold
      
      **Values:** `PERCENTAGE`, `ABSOLUTE`
    </ParamField>
    
    <ParamField body="threshold_value" type="string">
      Threshold value that triggers alerts
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="metadata" type="object">
  Custom metadata as key-value pairs
</ParamField>

## Feature Types Explained

<CardGroup cols={3}>
  <Card title="BOOLEAN" icon="toggle-on">
    Simple on/off feature flags. Used for access control without usage tracking.
    
    **Examples:**
    - Premium support
    - Advanced analytics
    - Custom branding
  </Card>
  
  <Card title="METERED" icon="gauge">
    Usage-based features that track consumption. Requires a meter for tracking.
    
    **Examples:**
    - API calls
    - Storage (GB)
    - Monthly active users
  </Card>
  
  <Card title="STATIC" icon="file-lines">
    Features with fixed configuration values (e.g., "5 team members", "10GB storage").
    
    **Examples:**
    - User seats
    - Storage limits
    - Project count
  </Card>
</CardGroup>

## Creating Features with Meters

For METERED features, you have two options:

<AccordionGroup>
  <Accordion title="Option 1: Use Existing Meter">
    Reference a meter you've already created:
    
    ```json
    {
      "name": "API Calls",
      "type": "METERED",
      "meter_id": "meter_xyz789"
    }
    ```
  </Accordion>
  
  <Accordion title="Option 2: Create Meter Inline">
    Define the meter configuration in the same request:
    
    ```json
    {
      "name": "API Calls",
      "type": "METERED",
      "meter": {
        "name": "API Call Counter",
        "event_type": "api_call",
        "aggregation": "COUNT"
      }
    }
    ```
    
    FlexPrice will create the meter automatically.
  </Accordion>
</AccordionGroup>

## After Creating a Feature

Once you've created a feature, you can:

1. **Add to plans**: Include the feature in your plan entitlements
2. **Check access**: Use the [Check Access](/api/features/check-access) endpoint to gate features
3. **Track usage**: Send events to meter usage for METERED features
4. **Set limits**: Configure usage limits via entitlements
