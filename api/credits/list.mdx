---
title: "List Credit Grants"
api: "GET /v1/credit-grants"
description: "Retrieve a list of credit grants with optional filtering by scope, plan, or subscription."
---

## Query Parameters

<ParamField query="scope" type="string">
  Filter by scope
  
  **Values:** `PLAN`, `SUBSCRIPTION`
</ParamField>

<ParamField query="plan_id" type="string">
  Filter by plan ID (only returns PLAN-scoped grants)
</ParamField>

<ParamField query="subscription_id" type="string">
  Filter by subscription ID (only returns SUBSCRIPTION-scoped grants)
</ParamField>

<ParamField query="cadence" type="string">
  Filter by cadence
  
  **Values:** `ONETIME`, `RECURRING`
</ParamField>

<ParamField query="expiration_type" type="string">
  Filter by expiration type
  
  **Values:** `NEVER`, `DURATION`, `BILLING_CYCLE`
</ParamField>

<ParamField query="limit" type="integer" default="10">
  Number of credit grants to return per page (max 100)
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Number of credit grants to skip for pagination
</ParamField>

<ParamField query="sort_by" type="string" default="created_at">
  Field to sort by
  
  **Values:** `created_at`, `updated_at`, `name`, `credits`
</ParamField>

<ParamField query="sort_order" type="string" default="desc">
  Sort order
  
  **Values:** `asc`, `desc`
</ParamField>

## Response

<ResponseField name="data" type="array">
  Array of credit grant objects
  
  <Expandable title="Credit grant object">
    <ResponseField name="id" type="string">
      Unique identifier for the credit grant
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Name of the credit grant
    </ResponseField>
    
    <ResponseField name="scope" type="string">
      Scope of the grant (PLAN or SUBSCRIPTION)
    </ResponseField>
    
    <ResponseField name="plan_id" type="string">
      ID of the plan (if scope is PLAN)
    </ResponseField>
    
    <ResponseField name="subscription_id" type="string">
      ID of the subscription (if scope is SUBSCRIPTION)
    </ResponseField>
    
    <ResponseField name="credits" type="string">
      Number of credits granted
    </ResponseField>
    
    <ResponseField name="cadence" type="string">
      Cadence of the grant (ONETIME or RECURRING)
    </ResponseField>
    
    <ResponseField name="period" type="string">
      Period for recurring grants (DAILY, WEEKLY, MONTHLY, etc.)
    </ResponseField>
    
    <ResponseField name="period_count" type="integer">
      Number of periods for recurring grants
    </ResponseField>
    
    <ResponseField name="expiration_type" type="string">
      How credits expire (NEVER, DURATION, or BILLING_CYCLE)
    </ResponseField>
    
    <ResponseField name="expiration_duration" type="integer">
      Duration before expiration (if expiration_type is DURATION)
    </ResponseField>
    
    <ResponseField name="expiration_duration_unit" type="string">
      Unit for expiration duration (DAYS, WEEKS, MONTHS, YEARS)

    <ResponseField name="start_date" type="string">
      Start date for the grant (subscription-scoped only)

    <ResponseField name="end_date" type="string">

    <ResponseField name="updated_at" type="string">
      Timestamp when grant was last updated
    </ResponseField>
  </Expandable>
</ResponseField>

## Common Filtering Patterns

<AccordionGroup>
  <Accordion title="View All Credits for a Customer">
    To see all credits available to a customer, you need to query both:
    1. Plan-scoped grants for any active subscriptions
    2. Subscription-scoped grants for their specific subscriptions
    
    ```python
    # Get customer's subscriptions
    subscriptions = client.subscriptions.list(customer_id="cust_123")
    
    # Get subscription-scoped grants
    for sub in subscriptions.data:
        grants = client.credit_grants.list(subscription_id=sub.id)
        
    # Get plan-scoped grants
    for sub in subscriptions.data:
        grants = client.credit_grants.list(
            scope="PLAN",
            plan_id=sub.plan_id
        )
    ```
  </Accordion>
  
  <Accordion title="Audit Recurring Credits">
    List all recurring credit grants to understand ongoing commitments:
    
    ```bash
    curl -X GET "https://api.flexprice.io/v1/credit-grants?cadence=RECURRING" \
      -H "Authorization: Bearer YOUR_API_KEY"
    ```
  </Accordion>
  
  <Accordion title="Find Expiring Credits">
    While the list endpoint doesn't directly filter by expiration date, you can:
    1. List all grants with `expiration_type=DURATION`
    2. Filter by application date ranges in your application logic
    3. Use the credit grant applications endpoint for scheduled credits
  </Accordion>
</AccordionGroup>
